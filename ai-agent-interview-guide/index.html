Module 01 — LLM Foundation

Transformer 核心架构

自注意力计算公式

Attention(Q,K,V) = softmax(QKT / sqrt(dk)) V

Q Query

当前 token 的"问题"

K Key

每个 token 的"标签"

V Value

每个 token 的"内容"

面试追问

为什么要除以 sqrt(dk)？防止点积过大导致 softmax 饱和区梯度消失

高频考点速记

Multi-Head Attention

多维度并行捕捉句法、语义、长距离依赖。降低单头计算量，提升泛化能力

RoPE 旋转位置编码

复数空间旋转注入位置信息，天然支持相对位置。LLaMA / Qwen / Mistral 均采用

GQA / MQA 优化

GQA：多个 Q 共享一组 KV，平衡精度与效率。MQA：所有 Q 共享一组 KV，极致省显存

FlashAttention

IO-aware 分块计算，减少 HBM 读写。无需近似，速度提升 2-4x

Module 01 — Efficient Fine-tuning

高效微调·推理优化

LoRA 低秩适配

冻结原始权重 W，仅训练低秩分解矩阵 A 和 B

W' = W + BA

r = 8~64，参数量仅为原模型 0.1%~1%

QLoRA

4-bit 量化 + LoRA，在消费级 GPU 上微调 65B 模型

推理优化全景

KV Cache

缓存已计算的 Key-Value，避免自回归生成中的重复计算。GQA/MQA 进一步降低 KV 显存

量化部署

FP16 → INT8 → INT4 / GPTQ / AWQ

Module 02 — RAG Advanced

混合检索·Rerank·幻觉处理

混合检索 = 向量 + BM25

向量检索语义相关（"SQL调优" ↔ "数据库优化"）
BM25精确匹配（"RFC 7231" 精确命中）

RRF 融合策略

score(d) = Σ 1/(k + rank(d))

k=60，排名越靠前贡献越高，比固定权重更稳定

Rerank 重排序

Bi-Encoder 检索 Top-20 → Cross-Encoder 精排 Top-5
召回率: 71% → 89%

幻觉处理六策略

1Prompt 约束：仅基于检索结果回答
2输出自校验：二次 LLM 检查依据
3引用标注：标注信息来源 chunk
4温度调低：0.1-0.3 降低随机性
5检索对齐：生成结果 vs 检索结果相似度
6兜底回答：低于阈值返回"未找到"

Agentic RAG

Agent 自主决定检索策略，不满意就换 query 重检。适合复杂场景，简单问答别过度设计

Module 03 — AI Agent

Agent = LLM 在循环中自主使用工具

Agent 四大核心模块：大脑 (LLM) + 规划 + 记忆 + 工具。与 LLM 的本质区别：LLM 只能"说"，Agent 能"做"

LLM vs Agent

记忆无记忆短期+长期记忆
工具不能调用API/搜索/数据库
规划被动响应自主拆解任务
方式一问一答思考→行动→观察循环

Agent 四大工作模式

ReActThought→Action→Observation 循环，最经典
Plan-Execute先规划再执行，Token 仅为 ReAct 1/5
Reflection完成后自我审查，双 Agent 评审
Multi-Agent多专业 Agent 协作，LangGraph/CrewAI

Workflow vs Agent

Workflow = 开发者控制流程，Token 约为 Agent 1/4
最佳实践：混合架构，整体 Workflow + 复杂环节 Agent

Module 04 — Prompt Engineering

Prompt 工程核心方法

Zero-Shot / Few-Shot

Zero-Shot直接提问，无示例。依赖模型预训练知识
Few-Shot给 2-5 个示例，引导输出格式和逻辑

Chain-of-Thought

让模型"逐步推理"而非直接输出答案

Few-Shot CoT: 示例中包含推理过程
Zero-Shot CoT: 添加 "Let's think step by step"

Self-Consistency

多次采样（temperature > 0），取多数投票结果。复杂推理任务准确率显著提升

DSPy 自动优化

Stanford 出品，用编译器思路自动搜索最优 Prompt。告别手工调参，让数据驱动 Prompt 优化

其他高频技巧

Role Playing 角色扮演 / System Prompt
Structured Output 结构化输出
ReAct Prompting / Tree-of-Thought
