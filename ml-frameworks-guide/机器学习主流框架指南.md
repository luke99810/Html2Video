## 1. 机器学习全景地图

在深入框架之前，先建立全局认知：

```
机器学习
├── 传统机器学习（结构化数据）
│   ├── 监督学习：分类、回归
│   ├── 无监督学习：聚类、降维
│   └── 集成方法：Boosting、Bagging
│
├── 深度学习（非结构化数据）
│   ├── 计算机视觉（CNN、ViT）
│   ├── 自然语言处理（RNN、Transformer）
│   ├── 生成模型（GAN、Diffusion、VAE）
│   └── 图神经网络（GNN）
│
└── 强化学习
    ├── 基于值（Q-Learning、DQN）
    └── 基于策略（PPO、SAC）
```

**框架与任务的对应关系：**

| 任务类型 | 首选框架 | 原因 |
|---------|---------|------|
| 表格数据分类/回归 | Scikit-learn + XGBoost/LightGBM | 成熟、高效、可解释 |
| 图像分类/检测 | PyTorch / TensorFlow | GPU加速、灵活网络定义 |

| 生产级部署 | TensorFlow Serving / TorchServe | 完善的部署工具链 |

## 2. NumPy & SciPy — 科学计算基石

### 为什么先学它？

几乎所有ML框架的底层都依赖NumPy。理解它是理解后续框架的前提。

### 核心概念

**ndarray — n维数组**

```python
import numpy as np

# 创建数组
a = np.array([1, 2, 3])           # 一维
b = np.zeros((3, 4))               # 3x4零矩阵
c = np.random.randn(3, 4)          # 3x4标准正态分布

# 关键属性
print(a.shape)    # (3,)    — 形状
print(a.dtype)    # int64   — 数据类型
print(a.ndim)     # 1       — 维度数

# 广播机制（Broadcasting）—— NumPy最重要的特性
# 不同形状的数组可以做运算，小形状自动"扩展"
a = np.array([[1], [2], [3]])  # (3, 1)
b = np.array([10, 20, 30])     # (3,)
c = a + b                       # (3, 3) — 自动广播
```

**矩阵运算 — ML的数学语言**

```python
# 点积（神经网络的核心操作）
W = np.random.randn(3, 4)   # 权重矩阵
x = np.random.randn(4)      # 输入向量
y = W @ x                    # 等价于 np.dot(W, x)

# 矩阵乘法的直觉：变换空间
# W: (3,4) × x: (4,) → y: (3,)
# 把4维输入映射到3维输出 —— 这就是全连接层做的事
```

**SciPy — 科学计算工具箱**

```python
from scipy import optimize, stats

# 优化（训练模型 = 最小化损失函数）
result = optimize.minimize(
    fun=lambda w: np.sum((X @ w - y) ** 2),  # 损失函数
    x0=np.zeros(X.shape[1])                   # 初始参数
)

# 统计检验
t_stat, p_value = stats.ttest_ind(group_a, group_b)
```

### 学习要点

- **ndarray的内存布局**：C顺序（行优先）vs Fortran顺序（列优先）
- **广播规则**：从尾部维度开始比较，1可以广播，维度必须相等或一方为1
- **避免Python循环**：用向量化操作替代，速度差100倍以上

## 3. Scikit-learn — 传统ML标杆

```python
model.fit(X_train, y_train)                                         # 2. 训练

```

### 数据处理Pipeline

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.compose import ColumnTransformer

# 数值列：填充 + 标准化
numeric_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='median')),
    ('scaler', StandardScaler())
])

# 分类列：填充 + OneHot
categorical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('encoder', OneHotEncoder(handle_unknown='ignore'))
])

# 组合
preprocessor = ColumnTransformer(transformers=[
    ('num', numeric_transformer, numeric_features),
    ('cat', categorical_transformer, categorical_features)
])

# 完整Pipeline
clf = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('classifier', RandomForestClassifier(n_estimators=100))
])

clf.fit(X_train, y_train)
```

### 模型选择与调参

### Functional API — 更灵活的模型构建

```python
# 多输入、多输出、分支网络
inputs = keras.Input(shape=(224, 224, 3))

# 共享特征提取
x = keras.layers.Conv2D(32, 3, activation='relu')(inputs)
x = keras.layers.MaxPooling2D()(x)
x = keras.layers.Conv2D(64, 3, activation='relu')(x)
x = keras.layers.GlobalAveragePooling2D()(x)

# 多任务输出
class_output = keras.layers.Dense(10, activation='softmax', name='class')(x)
reg_output = keras.layers.Dense(1, name='reg')(x)

model = keras.Model(inputs=inputs, outputs=[class_output, reg_output])

model.compile(
    optimizer='adam',
    loss={'class': 'sparse_categorical_crossentropy', 'reg': 'mse'},
    loss_weights={'class': 1.0, 'reg': 0.5}
)
```

### 自定义训练循环

```python
# 当标准fit()不够用时
@tf.function  # 编译为图模式，加速执行
def train_step(x, y):
    with tf.GradientTape() as tape:
        predictions = model(x, training=True)
        loss = loss_fn(y, predictions)
    
    gradients = tape.gradient(loss, model.trainable_variables)
    optimizer.apply_gradients(zip(gradients, model.trainable_variables))
    return loss

for epoch in range(epochs):
    for x_batch, y_batch in train_dataset:
        loss = train_step(x_batch, y_batch)
```

## 5. PyTorch — 研究者的首选

### 定位

Meta（Facebook）出品，动态图设计，Pythonic风格，学术界主流。

### 核心设计哲学

```
PyTorch vs TensorFlow 的根本区别：

TensorFlow 1.x:  先建图，再运行（静态图）
PyTorch:         边运行，边建图（动态图）

→ PyTorch更符合Python直觉，调试更自然
→ TensorFlow 2.x 也支持动态图（eager mode），但生态惯性在
```

### 张量操作

```python
import torch

# 创建
x = torch.tensor([1.0, 2.0, 3.0])       # 从数据
x = torch.randn(3, 4)                     # 随机
x = torch.zeros(2, 3)                     # 零张量

# GPU加速
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
x = x.to(device)

# 自动求导
x = torch.tensor([2.0], requires_grad=True)
y = x ** 2 + 3 * x
y.backward()
print(x.grad)  # tensor([7.])  → dy/dx = 2x + 3 = 7
```

### 构建模型：nn.Module

```python
import torch.nn as nn

class NeuralNetwork(nn.Module):
    def __init__(self):
        super().__init__()
        self.flatten = nn.Flatten()
        self.linear_relu_stack = nn.Sequential(
            nn.Linear(784, 128),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(64, 10)
        )
    
    def forward(self, x):
        x = self.flatten(x)
        logits = self.linear_relu_stack(x)
        return logits

model = NeuralNetwork().to(device)
print(model)  # 自动打印模型结构
```

### 标准训练循环

```python
# PyTorch训练循环是"显式"的——每一步都写清楚
# 这是PyTorch的学习曲线稍陡，但灵活性极高的原因

criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(epochs):
    # --- 训练阶段 ---
    model.train()
    for X_batch, y_batch in train_loader:
        X_batch, y_batch = X_batch.to(device), y_batch.to(device)
        
        # 前向传播
        pred = model(X_batch)
        loss = criterion(pred, y_batch)
        
        # 反向传播
        optimizer.zero_grad()   # 清空梯度（重要！）
        loss.backward()         # 计算梯度
        optimizer.step()        # 更新参数
    
    # --- 验证阶段 ---
    model.eval()
    val_loss = 0
    correct = 0
    with torch.no_grad():      # 不计算梯度
        for X_batch, y_batch in val_loader:
            X_batch, y_batch = X_batch.to(device), y_batch.to(device)
            pred = model(X_batch)
            val_loss += criterion(pred, y_batch).item()
            correct += (pred.argmax(1) == y_batch).sum().item()
    
    print(f"Epoch {epoch}: val_acc={correct/len(val_dataset):.4f}")
```

### 数据加载

```python
from torch.utils.data import Dataset, DataLoader

class CustomDataset(Dataset):
    def __init__(self, features, labels):
        self.features = torch.tensor(features, dtype=torch.float32)
        self.labels = torch.tensor(labels, dtype=torch.long)
    
    def __len__(self):
        return len(self.labels)
    
    def __getitem__(self, idx):
        return self.features[idx], self.labels[idx]

train_dataset = CustomDataset(X_train, y_train)
train_loader = DataLoader(
    train_dataset,
    batch_size=32,
    shuffle=True,           # 训练集打乱
    num_workers=4,          # 多进程加载
    pin_memory=True         # 加速GPU传输
)
```

### PyTorch vs TensorFlow 对比

| 维度 | PyTorch | TensorFlow/Keras |
|-----|---------|------------------|
| 图模式 | 动态图（默认） | 静态图优先，支持动态 |
| 调试 | Python原生调试 | 较难调试 |
| 模型定义 | 继承nn.Module | Sequential / Functional |
| 训练循环 | 手写（显式控制） | model.fit()（隐式封装） |
| 学术论文 | ~80%使用 | ~20%使用 |
| 工业部署 | TorchServe（发展中） | TF Serving（成熟） |
| 学习曲线 | 中等偏陡 | 入门更容易 |
| 灵活性 | ★★★★★ | ★★★★☆ |

## 6. Keras — 高层API快速上手

### 定位

深度学习的"快速入门"接口。现在是TensorFlow的官方高层API，也有独立的多后端版本（Keras 3）。

### 三种模型构建方式

```python
# 方式1：Sequential —— 线性堆叠，最简单
model = keras.Sequential([
    keras.layers.Dense(64, activation='relu'),
    keras.layers.Dense(10, activation='softmax')
])

# 方式2：Functional API —— 有分支/多输入/多输出
inputs = keras.Input(shape=(784,))
x = keras.layers.Dense(64, activation='relu')(inputs)
outputs = keras.layers.Dense(10, activation='softmax')(x)
model = keras.Model(inputs, outputs)

# 方式3：子类化 —— 最大灵活性
class MyModel(keras.Model):
    def __init__(self):
        super().__init__()
        self.dense1 = keras.layers.Dense(64, activation='relu')
        self.dense2 = keras.layers.Dense(10, activation='softmax')
    
    def call(self, x):
        x = self.dense1(x)
        return self.dense2(x)
```

### 常用回调

```python
callbacks = [
    # 早停：验证集不提升就停止
    keras.callbacks.EarlyStopping(
        monitor='val_loss',
        patience=5,
        restore_best_weights=True
    ),
    # 学习率衰减

        monitor='val_loss',
        factor=0.5,

]

model.fit(X_train, y_train, epochs=100, callbacks=callbacks, validation_split=0.2)
```

## 7. Hugging Face Transformers — NLP新范式

### 定位

预训练模型生态中心。一行代码加载BERT、GPT、LLaMA等模型，NLP领域的"App Store"。

### 核心理念

```
传统ML：  手工特征 → 训练模型
深度学习：  网络架构 → 从头训练
预训练时代： 预训练模型 → 微调（Fine-tune）→ 下游任务

Hugging Face 让第三步变得极其简单
```

### Pipeline — 5行代码做NLP

```python
from transformers import pipeline

# 情感分析
classifier = pipeline("sentiment-analysis")
result = classifier("这个产品太棒了！")
# [{'label': 'POSITIVE', 'score': 0.9998}]

# 文本生成
generator = pipeline("text-generation", model="gpt2")
result = generator("Once upon a time", max_length=50)

# 问答
qa = pipeline("question-answering")
result = qa(question="谁发明了Python？", context="Python由Guido van Rossum创建")

# 翻译
translator = pipeline("translation_en_to_zh")
result = translator("Hello, how are you?")

# 命名实体识别
ner = pipeline("ner")
result = ner("Apple was founded by Steve Jobs in California")
```

### 微调预训练模型

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification, Trainer, TrainingArguments
from datasets import load_dataset

# 1. 加载预训练模型和分词器
model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=2)

# 2. 加载数据集
dataset = load_dataset("csv", data_files="my_data.csv")

# 3. 分词
def tokenize(batch):
    return tokenizer(batch['text'], padding=True, truncation=True, max_length=128)

tokenized_dataset = dataset.map(tokenize, batched=True)

# 4. 训练配置
training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    learning_rate=2e-5,
    weight_decay=0.01,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True
)

# 5. 训练
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_dataset["train"],
    eval_dataset=tokenized_dataset["test"],
    tokenizer=tokenizer
)

trainer.train()

# 6. 保存
model.save_pretrained("./my-finetuned-model")
tokenizer.save_pretrained("./my-finetuned-model")
```

## 8. XGBoost / LightGBM / CatBoost — 表格数据三剑客

### 为什么深度学习不是万能的？

```
数据类型          最佳选择
─────────────────────────────
图像/视频/音频    → 深度学习（CNN/ViT）
文本             → Transformer
─────────────────────────────
表格/结构化数据    → GBDT（这三位）  ← 深度学习在这里反而不如
```

Kaggle竞赛中，表格数据的冠军方案几乎都是GBDT的变体。

### XGBoost — 正则化Boosting

```python
import xgboost as xgb

# 原生API（更精细控制）
dtrain = xgb.DMatrix(X_train, label=y_train)
dtest = xgb.DMatrix(X_test, label=y_test)

params = {
    'objective': 'binary:logistic',
    'max_depth': 6,
    'learning_rate': 0.1,
    'subsample': 0.8,
    'colsample_bytree': 0.8,
    'eval_metric': 'auc'
}

model = xgb.train(
    params, dtrain,
    num_boost_round=500,
    evals=[(dtest, 'eval')],
    early_stopping_rounds=30,
    verbose_eval=50
)

# Scikit-learn兼容API（更方便Pipeline集成）
from xgboost import XGBClassifier
model = XGBClassifier(
    n_estimators=500,
    max_depth=6,
    learning_rate=0.1,
    subsample=0.8,
    early_stopping_rounds=30
)
model.fit(X_train, y_train, eval_set=[(X_test, y_test)], verbose=False)
```

### LightGBM — 速度之王

```python
import lightgbm as lgb

train_data = lgb.Dataset(X_train, label=y_train)
val_data = lgb.Dataset(X_test, label=y_test, reference=train_data)

params = {
    'objective': 'binary',
    'metric': 'auc',
    'learning_rate': 0.05,
    'num_leaves': 31,           # LightGBM用叶子数而非深度控制
    'feature_fraction': 0.8,
    'bagging_fraction': 0.8,
    'bagging_freq': 5,
    'verbose': -1
}

model = lgb.train(
    params, train_data,
    num_boost_round=1000,
    valid_sets=[val_data],
    callbacks=[
        lgb.early_stopping(30),
        lgb.log_evaluation(50)
    ]
)

# Scikit-learn兼容
from lightgbm import LGBMClassifier
model = LGBMClassifier(n_estimators=1000, num_leaves=31, learning_rate=0.05)
model.fit(X_train, y_train, eval_set=[(X_test, y_test)])
```

### CatBoost — 类别特征专家

```python
from catboost import CatBoostClassifier, Pool

# CatBoost的杀手锏：自动处理类别特征，无需OneHot
cat_features = [0, 2, 5]  # 类别列的索引

train_pool = Pool(X_train, y_train, cat_features=cat_features)

    loss_function='Logloss',

model.fit(train_pool, eval_set=val_pool)
```

## 10. 实战：同一任务，多框架对比

**任务：MNIST手写数字分类**

### Scikit-learn

```python
from sklearn.linear_model import SGDClassifier
from sklearn.datasets import fetch_openml

X, y = fetch_openml('mnist_784', version=1, return_X_y=True)
X_train, X_test = X[:60000] / 255.0, X[60000:] / 255.0
y_train, y_test = y[:60000], y[60000:]

model = SGDClassifier(loss='hinge', max_iter=20)
model.fit(X_train, y_train)
print(f"准确率: {model.score(X_test, y_test):.4f}")
# ~88% — 传统ML的天花板
```

### Keras (TensorFlow)

```python
import tensorflow as tf

(X_train, y_train), (X_test, y_test) = tf.keras.datasets.mnist.load_data()
X_train, X_test = X_train / 255.0, X_test / 255.0

model = tf.keras.Sequential([
    tf.keras.layers.Flatten(input_shape=(28, 28)),
    tf.keras.layers.Dense(128, activation='relu'),
    tf.keras.layers.Dropout(0.2),
    tf.keras.layers.Dense(10, activation='softmax')
])

model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.fit(X_train, y_train, epochs=5, validation_split=0.1)
# ~98% — 简单神经网络
```

### PyTorch

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

X_train_t = torch.tensor(X_train.reshape(-1, 784), dtype=torch.float32)
y_train_t = torch.tensor(y_train, dtype=torch.long)
train_loader = DataLoader(TensorDataset(X_train_t, y_train_t), batch_size=64, shuffle=True)

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(784, 128), nn.ReLU(), nn.Dropout(0.2),
            nn.Linear(128, 10)
        )
    def forward(self, x):
        return self.net(x)

model = Net()
optimizer = torch.optim.Adam(model.parameters())
criterion = nn.CrossEntropyLoss()

for epoch in range(5):
    for X_b, y_b in train_loader:
        loss = criterion(model(X_b), y_b)
        optimizer.zero_grad(); loss.backward(); optimizer.step()
# ~98% — 同等架构同等级别
```

| 框架 | 代码量 | 准确率 | 灵活性 | 上手难度 |

| Keras | 10行 | ~98% | ★★★☆☆ | ★★☆☆☆ |
| PyTorch | 20行 | ~98% | ★★★★★ | ★★★☆☆ |

### 常见陷阱

1. **跳过数学直接调包** — 理解损失函数、梯度下降的原理，才知道为什么模型不收敛
2. **只学框架不学数据** — 80%的工作在数据清洗和特征工程，不是调参
3. **追新框架忽视基础** — Scikit-learn的思维（Pipeline、CV、评估）是所有ML工作的根基
4. **不看官方文档** — 框架更新快，博客教程容易过时，官方文档最可靠
5. **只跑示例不动手改** — 修改网络结构、换数据集、调超参，才是真正学会

---

> **最后的建议：** 不要试图同时学所有框架。选一个深度学习框架（推荐PyTorch）+ Scikit-learn + 一个GBDT库，这三个覆盖90%的场景。其余的在工作中按需学习。
