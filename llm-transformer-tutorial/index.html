01 / TOKENIZATION

Token：模型的"字母表"

计算机不懂文字，它只懂数字。Token 化就是建立文字↔数字的映射

BPE 分词算法（Byte Pair Encoding）
原始文本 "unhappiness"
初始拆分 un ##hap ##pi ##ness
Token ID [8472, 4251, 3892, 1567]
为什么需要 Token 化？
1. 压缩文本："un-"前缀只需一个数字
2. 处理生僻词：通过子词组合表示
3. 统一长度：固定维度便于并行计算
Token 嵌入（Embedding）
# 每个 Token 对应一个高维向量 Token "hello" → ID 15496 → Vector[768] # 向量中的每个数字代表语义特征 [0.23, -0.15, 0.89, ..., 0.04] ↓ ↓ ↓ ↓ 情感 时态 主题相关性 语法角色

02 / TRANSFORMER ARCHITECTURE

Transformer：注意力就是一切

摒弃循环，用注意力机制并行处理整个序列

核心计算：Query · Key = Attention
# 自注意力机制（Self-Attention） for each token in sequence: Q = W_q @ token_embedding # 查询：我要找什么？ K = W_k @ token_embedding # 键：我有什么？ V = W_v @ token_embedding # 值：实际内容 scores = Q @ K.T / sqrt(dim) # 相似度分数 weights = softmax(scores) # 归一化为概率 output = weights @ V # 加权求和
Query (查询)
[0.2, -0.5, 0.8]
Key (键)
[0.3, 0.1, -0.4]
Attention Score
0.2×0.3 + (-0.5)×0.1 + 0.8×(-0.4) = -0.27
Attention(Q,K,V) = softmax(QKT/√dk)V
位置编码（Positional Encoding）
Transformer 没有循环结构，需要显式告诉模型每个词的位置信息
# 正弦位置编码公式 PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model)) # 实际效果：每个位置有独特"指纹"

Embedding + Positional Encoding = 模型输入
每个 Token 的向量 = 语义信息 + 位置信息
Transformer 结构
多头注意力
8-96 个注意力头并行计算，每个关注不同特征（语法、语义、指代等）
Add & Norm
残差连接 + Layer Normalization，稳定深层网络训练
前馈网络 (FFN)
对每个位置独立应用全连接层，增加非线性表达能力
Add & Norm
再次残差连接，堆叠 12-96 层形成深度网络
GPT 只用 Decoder
• Encoder：双向注意力（理解）
• Decoder：单向注意力（生成）
• GPT 是自回归生成模型

05 / RLHF ALIGNMENT

RLHF：让模型对齐人类价值观

预训练学的是"像人说话"，RLHF 学的是"说人想听的话"

RLHF 三阶段
1
SFT - 监督微调
用高质量人工编写的问答对继续训练，让模型学会对话格式和基本指令遵循。 数据量：1万-10万条高质量对话
2
Reward Model - 奖励模型
人类标注者对同一问题的多个回答进行排序，训练奖励模型学习人类偏好。 模型学会：什么回答更好
3
PPO - 强化学习优化
用奖励模型作为"裁判"，通过 PPO 算法优化策略模型，生成更高分的回答。 目标：最大化奖励，同时不要偏离原始模型太远
奖励模型工作原理
回答 A（更好）
Transformer 是一种神经网络架构，核心是自注意力机制。它通过计算序列中每个位置与其他位置的关联度来捕捉上下文信息。
Reward: +2.5
回答 B（较差）
就是一种深度学习模型，用来处理自然语言任务。效果很好，被广泛应用。
Reward: -0.8
奖励模型评判维度
• 有用性：是否回答了问题
• 真实性：内容是否准确
• 无害性：是否安全、无偏见
• 流畅性：表达是否自然
PPO 算法简述
# PPO (Proximal Policy Optimization) for iteration in range(N): # 1. 生成回答 response = policy_model.generate(prompt) # 2. 奖励模型打分 reward = reward_model(prompt, response) # 3. 计算优势函数 advantage = reward - baseline # 4. 策略更新（带裁剪） loss = -min(ratio × advantage, clip(ratio) × advantage) # 5. 同时加 KL 惩罚，防止模型跑偏 loss += kl_penalty(policy, reference)
为什么不用纯强化学习？
• 奖励模型不完美，可能钻空子
• KL 惩罚保证不偏离原始能力
• 组合多种方法更稳定
